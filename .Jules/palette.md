## 2024-05-08 - Enforce native validation and ARIA roles for robust forms
**Learning:** For forms in this project (like `SubmitForm.jsx`), relying solely on JavaScript validation with `e.preventDefault()` can be poor for accessibility. Essential requirements must be declared natively (using `required` and `aria-required="true"`) to allow screen readers and the browser to properly enforce and announce them. Additionally, dynamically rendered notifications must have `role="alert"` or `role="status"` to be announced by screen readers without focus changes.
**Action:** Always include `required` and `aria-required="true"` on mandatory `<input>`, `<select>`, and `<textarea>` elements. Use `role="alert"` for error messages and `role="status"` for success messages. Remember to add `aria-hidden="true"` to decorative indicator characters like `*`.
