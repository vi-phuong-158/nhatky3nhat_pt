## 2024-05-18 - Native Validation & Semantic ARIA Roles in Forms
**Learning:** React state-driven forms often skip native browser validation attributes and lack appropriate ARIA roles. While custom Javascript validation provides flexibility, omitting attributes like `required` and `aria-required="true"` bypasses native screen-reader cues for mandatory inputs. Additionally, dynamic success/error notification regions are not announced effectively to assistive technologies unless designated with `role="alert"` (for errors) or `role="status"` (for success messages). Lastly, decorative icons in forms and headers unnecessarily clutter screen reader outputs if not explicitly hidden using `aria-hidden="true"`.
**Action:** Always enforce native validation cues (e.g. `required`, `aria-required="true"`) alongside custom validation logic. Wrap form feedback notifications in appropriate semantic ARIA roles (`role="alert"` or `role="status"`). Apply `aria-hidden="true"` to strictly decorative icon spans (like `material-symbols-outlined`) to streamline assistive technology experience.
