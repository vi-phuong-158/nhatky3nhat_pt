## 2024-05-15 - Dynamic Form Validation and Alerts A11y
**Learning:** Custom form validations and dynamic alerts (like error/success messages in `SubmitForm.jsx`) are visually clear but invisible to screen readers without ARIA live regions. Using `aria-live="assertive"` combined with `role="alert"` for errors ensures immediate feedback. Using `aria-live="polite"` combined with `role="status"` for success messages provides feedback without interrupting the user's flow. Additionally, fields visually marked as required (e.g., with an asterisk) must have `aria-required="true"` or the `required` attribute so screen readers announce them as mandatory.
**Action:** Always verify that dynamic feedback (toasts, alerts, inline errors) has appropriate `aria-live` regions and `role` attributes, and ensure visually required fields have corresponding programmatic required attributes.
