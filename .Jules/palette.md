## 2025-02-28 - Add native validation and alert roles to Javascript-driven form
**Learning:** For accessibility, forms relying on JavaScript validation must still enforce native HTML constraints (`required`, `aria-required="true"`) on input elements to ensure screen readers announce them properly. Additionally, dynamic success/error notification blocks must have appropriate ARIA roles (`role="alert"` for errors, `role="status"` for non-critical success messages) to ensure they are announced when they appear. Decorative icon elements, such as material-symbols, should use `aria-hidden="true"` so they aren't incorrectly read as their font ligature text.
**Action:** When creating forms, add native `required` and `aria-required="true"` to mandatory inputs, use `role="alert"` and `role="status"` on dynamic message divs, and ensure decorative icons use `aria-hidden="true"`.
