## 2024-05-14 - Add Native Form Validation and ARIA patterns to SubmitForm
**Learning:** Decorative icons (like Google Material Symbols) rendered as text nodes (e.g. `person`, `call`, `apartment`) inside forms can be read out by screen readers if they lack `aria-hidden="true"`, causing confusion. Furthermore, custom forms that rely entirely on JavaScript validation provide poor UX for keyboard and assistive tech users without native HTML5 `required` and `aria-required="true"` attributes. Notifications also need `role="alert"` or `role="status"` to ensure screen readers announce them dynamically.
**Action:** When creating forms, always pair visual requirements (like a red `*`) with `required aria-required="true"`. Use `aria-hidden="true"` on purely decorative icons. Ensure dynamic notifications use `role="alert"` or `role="status"`.
